## Flower Exchange

This project is a simple matching/exchange app with:

- A Node.js REST API server (`api-server`) that processes orders sequentially and generates execution reports.
- A React frontend (`frontend`) that lets you submit orders manually or upload a CSV.

The API runs on `http://localhost:8080`.

## Prerequisites

- Node.js (includes `npm`)

## Run the API Server

1. Open a terminal in the project root: `Flower-exchange`
2. Start the API:

```powershell
cd api-server
npm install
npm run dev
```

`npm run dev` uses `nodemon` and restarts automatically.

## Run the Frontend

1. Open a second terminal
2. Start the UI:

```powershell
cd frontend
npm install
npm run dev
```

Then open the URL shown by Vite (typically `http://localhost:5173`).

## Input Orders (API)

### 1) Submit JSON orders

Endpoint: `POST http://localhost:8080/api/orders`

Body: either a single order object or an array of orders.

Order fields:

- `clientOrderId` (string): 1-7 alphanumeric uppercase characters
- `instrument` (string): one of `Rose`, `Lavender`, `Lotus`, `Tulip`, `Orchid` (case-insensitive)
- `side` (number): `1` = Buy, `2` = Sell
- `quantity` (number): integer between `10` and `1000`, must be a multiple of `10`
- `price` (number): must be strictly greater than `0`

Example (PowerShell / `curl`):

```powershell
curl -X POST http://localhost:8080/api/orders `
  -H "Content-Type: application/json" `
  -d '[{"clientOrderId":"aa13","instrument":"Rose","side":2,"quantity":100,"price":55.0}]'
```

### 2) Submit CSV

Endpoint: `POST http://localhost:8080/api/orders/csv`

- Content type: `multipart/form-data`
- Form field name: `file`

CSV header (the API expects these column names):

- `Cl.Ord.ID` (also supports `Cl. Order ID` and `Cl.Ord.ID`)
- `Instrument`
- `Side`
- `Quantity`
- `Price`

Example CSV first row:

```csv
Cl.Ord.ID,Instrument,Side,Quantity,Price
```

Use the frontend (tab “Upload CSV”) or Postman/curl to upload the file.

## Execution Reports Output

Execution reports statuses:

- `New`
- `Fill`
- `PFill`
- `Rejected`

To fetch the most recent execution reports generated by the latest API call:

Endpoint: `GET http://localhost:8080/api/reports/latest`

Response: an array of report objects like:

- `orderId`
- `clientOrderId`
- `instrument`
- `side`
- `execStatus`
- `quantity`
- `price`
- `transactionTime`

## Clear State (testing)

Endpoint: `POST http://localhost:8080/api/reports/clear`

This clears the in-memory order book and report history.
